# 第I部 プレイヤー向けルールブック

## 1. ゲームの概要

本ゲームは、プレイヤーが学習塾の「室長」となり、夏期講習を控えた **通常期(5月下旬〜7月上旬の4ターン)** と、夏期講習本番である **講習期(前期3日+中期2日+後期4日の9ターン)** からなる **全13ターン** の教室運営を行うシミュレーションゲームである。前作「春期講習編」の続編であり、テーマは以下の2点に集約される。

- **多忙な教室運営**: 通常期の授業運営と並行して、夏期講習の準備(=スタッフ別デッキの構築)を進める。
- **スタッフを育成する**: 通常期で各スタッフに割り当てたカードがそのスタッフ専用の「講習デッキ」として蓄積され、講習期の戦力となる。

| 項目 | 仕様 |
|---|---|
| プレイ人数 | ルールブック上は **1〜6人**、ブラウザ版実装は **ソロ専用** (A1-1) |
| プレイ時間 | 30分目安 (A1-2) |
| 難易度 | FRESH / PRO の2種類(春期から継承) |
| カードプール | 春期と同一(`cards_fresh.csv` / `cards_pro.csv`を夏期リポジトリへコピーして流用、A14-3) |
| 総ターン数 | 13(通常期4 + 講習期9) |

## 2. 構成要素

### 2.1 スタッフ

- **室長** ・ **専任講師(以下「講師」)** ・ **事務** の3名(春期準拠)
- 講習期では別途 **アルバイト講師** が4人目のスタッフ枠として稼働する(通常期は3名、講習期は4名)

### 2.2 ステータス(春期と同じ4指標)

| ステータス | 初期値(FRESH) | 初期値(PRO) | カテゴリ対応 |
|---|---|---|---|
| 体験 | 0 | 0 | 動員 |
| 入塾 | 0 | 0 | 教務 |
| 満足 | 3 | 3 | 応対 |
| 経理 | 3 | 5 | 庶務 |

※ ランクCSVは夏期用に新設する。当面の閾値は春期講習の `rankFresh.csv` / `rankPro.csv` の値を仮利用し、後日夏期用に改訂する(A12-2, A12-3)。

### 2.3 カード

- **レアリティ**: N / R / SR / SSR
- **カテゴリ**: 動員 / 教務 / 応対 / 庶務(春期準拠)
- **スタッフ制限**: 効果テキスト【】内に許可スタッフを列挙(春期準拠)
- **効果テキスト書式**: 春期準拠(`<>`条件、`+X`値変化、特殊効果キーワード)

### 2.4 デッキ群

| デッキ名 | 所有者 | 構成タイミング | 内容 |
|---|---|---|---|
| 8山札(研修プール) | 全プレイヤー共有(引いた結果は秘匿) | ゲーム開始時 | 4カテゴリ × 2プール(地域/全校) |
| Nレアリティ補完プール | プレイヤー個別 | ゲーム開始時 | 5枚のNカード(問合対応の基本×2 + 質問対応の基本 + 生徒面談の基本 + 経理精算の基本) |
| スタッフ別講習デッキ × 3 | プレイヤー個別 | 通常期教室会議で蓄積 | 通常期で各スタッフに割り当てたカード(N含む) |
| アルバイト講師デッキ | プレイヤー個別 | 通常期教室会議で1枚スロット | R/SRかつ講師可カード(最新1枚のみ) |

**公開ポリシー**: スタッフ別講習デッキ・アルバイト講師デッキは **常時公開**(A6-8)。研修フェーズで引いた手札のみ秘匿。

### 2.5 トークン(特殊効果用、PRO中心)

- **情熱** / **発想** / **疲労** / **整理** / **並行**(春期から効果内容変更、A10-1〜A10-5)
- **講習期トークン**(PRO限定): 情熱トークン / 発想トークン / 整理トークン(A11-1〜A11-7)

## 4. ゲームの準備

1. 各プレイヤーはプレイヤーボード・3スタッフ枠・空のスタッフ別講習デッキ × 3・空のアルバイト講師デッキを受け取り、ステータスを初期値にセットする。
2. **Nレアリティ補完プール 5枚** を各プレイヤーに配布:
   - 問合対応の基本(動員) × 2
   - 質問対応の基本(教務) × 1
   - 生徒面談の基本(応対) × 1
   - 経理精算の基本(庶務) × 1
3. **8山札(研修プール)** をゲーム開始時に構成する:
   - 4カテゴリ × {地域研修プール, 全校研修プール} = 8山
   - **地域研修プール**(各カテゴリ): Rカード全て + 当該カテゴリSRの半数
   - **全校研修プール**(各カテゴリ): SSRカード全て + 当該カテゴリSRの残り半数
   - SRの内訳(どのSRが地域/全校に入るか)は **毎ゲームシャッフル**:
     - 各カテゴリのSRをカテゴリ内で半数ずつランダム分配(端数は地域寄せ)
     - 山札内の並び順もシャッフル
4. 8山をシャッフルして伏せ、テーブル中央に並べる。多人数の場合は全員で共有する。
5. PROのみ: 春期準拠の研修リフレッシュ残回数を初期化する。

## 5. 通常期(全4ターン)

各ターンは3フェーズで進行する。各ターンの「月下旬/月上旬」割当と引くプールの対応は以下のとおり。

| ターン | 月 | 引くプール |
|---|---|---|
| 1 | 5月下旬 | 地域研修 |
| 2 | 6月上旬 | 全校研修 |
| 3 | 6月下旬 | 地域研修 |
| 4 | 7月上旬 | 全校研修 |

### 5.1 研修フェーズ

そのターンに引くプール(地域 or 全校)の **4カテゴリ4山** が対象。

1. プレイヤーは「**4カテゴリのうち1つを選んで1枚カードを引く**」を **4回** 繰り返す。
   - 同じカテゴリを連続選択することも可能(回数制限なし、A4-1)
   - 各回ドロー後に引いたカードを確認してから、次のカテゴリを選択できる
   - 引いたカードは **自分のみ確認**(多人数時、他プレイヤーには公開しない)
2. 4回終了時点で **手札4枚** が形成される。
3. 引いたカードはそのプレイヤーの手札に加わる。山が枯渇した場合、その山の捨て札をシャッフルして再構成する。
4. **多人数時**: 8山札は **全員共有**だが、各プレイヤーが引いたカードは **手札に伏せて保持** し、他プレイヤーには見えない(A3-6)。

### 5.2 教室行動フェーズ

1. プレイヤーは手札4枚から3枚を選び、3スタッフ(室長/講師/事務)に **1枚ずつ** 配置する。
   - 1スタッフに1枚が基本(並行効果カードのみ追加配置可)
   - 確定までは置き換え自由
   - スタッフ制限(【室長】等)は春期準拠
2. **未割当スタッフ**(プレイヤーが配置しなかったスタッフ)には、別途用意してある **Nレアリティ補完プール** から **1枚を選んで割り当て、効果を発揮させる**。
   - Nカードの選び順はプレイヤー任意
   - Nカードには職種制限はなく、未割当スタッフであれば任意のスタッフへ割り当ててよい(A5-10)。
   - Nプールが枯渇している場合、未割当スタッフには何も割り当てない(A5-9)。
3. **おすすめ行動ボーナス(通常期)**:
   - **動員カテゴリ** のカード1枚使用につき **体験+2**
   - **庶務カテゴリ** のカード1枚使用につき **経理+2**
   - 上記はカードの本来効果に加算される
4. 全プレイヤーが配置を完了したら、スタートプレイヤーから順にカード効果を解決する(春期準拠)。
5. ステータス変動・特殊効果の発動は春期準拠(`§7` 特殊効果一覧 参照)。

### 5.3 教室会議フェーズ

順序は以下のとおり。

1. **スタッフ枠に配置されたカード(N含む)** → 配置先スタッフの **講習デッキ** に移動。
   - **Nレアリティ補完プールから割り当てられたNカードも、その配置先スタッフの講習デッキへ移動する**(A5-7)。これによりNプールは消費されていく(N枯渇については §23 Q5-9 参照)。
2. **割り当てなかった手札(余りカード、通常1枚以上)** について:
   - 以下の **両条件** を満たす場合、プレイヤーは **アルバイト講師デッキ** へ移動できる(任意):
     - レアリティが **R または SR**
     - 効果テキスト【】に「講師」が含まれる、または【】指定がない(全スタッフ可)カード
   - 既にアルバイト講師デッキに別カードがある場合、元のカードは **ゲームから除外**(1枚スロット)。
   - 余りカードが複数枚あるとき、プレイヤーは1枚を選んでアルバイト講師デッキ候補とし、他は全てゲーム除外する。
3. 通常期教室会議では、春期にあった「**任意削除**」や「**手札補充ドロー**」は実施しない。

## 6. 講習期(全9ターン)

| ターン | 名称 | 準備フェーズ | 教室行動 | 教室会議 |
|---|---|---|---|---|
| 5 | 前期1日目 | 開始前にまとめて **3回** | あり | あり |
| 6 | 前期2日目 | なし | あり | あり |
| 7 | 前期3日目 | なし | あり | あり |
| 8 | 中期1日目 | なし | あり | あり |
| 9 | 中期2日目 | なし | あり | あり(**SSR復活判定**) |
| 10 | 後期1日目 | 開始前にまとめて **2回** | あり | あり |
| 11 | 後期2日目 | なし | あり | あり |
| 12 | 後期3日目 | なし | あり | あり |
| 13 | 後期4日目 | なし | あり | あり(ゲーム終了) |

### 6.1 講習期準備フェーズ(前期1日目開始前 / 後期1日目開始前のみ)

1. プレイヤーは自分の **3スタッフ別講習デッキ + アルバイト講師デッキ** の計4デッキを公開状態で並べる(常時公開なので状態維持)。
2. **「8山札から1山を選び、トップ3枚を引き、その中から1枚を選んで、4デッキいずれかの同一レアリティ以下の既存カード1枚と交換」** を以下の回数連続実行:
   - 前期1日目開始前: **3回**
   - 後期1日目開始前: **2回**
3. 各回の手順:
   1. プレイヤーは **通常期研修で使った8山札**(継続使用) から **1山** を選び、**トップ3枚** を見る(A7-8: 春期式で確定)。
   2. 3枚から1枚を選び「新カード」とする。残り2枚はその山の捨て札に戻す。
   3. 「新カード」について、以下のいずれかを実行:
      - **交換**: 3スタッフ別 + アルバイト講師デッキの計4デッキから、新カードと **同一レアリティ以下** の既存カード1枚を選び、新カードと交換する。押し出された既存カードは **ゲームから除外**。
      - **交換を放棄**: 新カードを **そのままゲームから除外**。
4. 山が枯渇したら捨て札を再シャッフル。

### 6.2 講習期教室行動フェーズ

1. プレイヤーは **3スタッフ別講習デッキ + アルバイト講師デッキの計4デッキ** それぞれから、**表向きのカード1枚** を選んで配置する:
   - 各デッキから1枚ずつ、計 **4枚/ターン** が基本
   - **並行効果**を持つカードを追加で配置することは可能(同一スタッフに重ね配置)
   - 配置しない(=休む)選択も可
2. 配置したカードの効果を解決する:
   - **おすすめ行動ボーナス(講習期)**:
     - **教務カテゴリ** のカード1枚使用につき **入塾+2**
     - **応対カテゴリ** のカード1枚使用につき **満足+2**
   - ステータス変動は春期と同じ計算式(体験→入塾の制約を含む)
3. **使用されたSR / SSRカード** はデッキ内で **裏返し** になり、以降使用不可となる:
   - Rカード・Nカードは何度でも使用可
   - 同名SR/SSRが複数あるときは個別管理(instanceIdベース)
4. **講習期にNカード補完はない**(Nプールはターン5以降は参照しない)。

### 6.3 講習期教室会議フェーズ

以下の処理を順に行う。

1. **SR復活判定** (各ターン):
   - そのターンに **1枚もカードを使用しなかったスタッフ** のデッキ内の **裏返しSRがすべて表向き** に戻る。
2. **SSR復活判定** (中期2日目=ターン9の教室会議でのみ判定):
   - **中期(ターン8とターン9)の両ターンで1枚もカードを使わなかったスタッフ** のデッキ内の **裏返しSSRがすべて表向き** に戻る (A2-2の「中期全ターン=2日」を反映)。
3. **PRO限定 — 講習期トークン消費アクション**: `§8` 参照。

復活したSR/SSRは再度使用後また裏返る(同一ルール)。

通常期に獲得した講習期トークンは消滅せず、講習期教室会議フェーズで消費する(A9-6, A11-3)。疲労は情熱トークンを減少させる。

## 8. PRO限定 ― 講習期トークン

### 8.1 獲得

- 情熱効果1回につき **情熱トークン+1**(A10-2, A11-1)
- 発想効果1回につき **発想トークン+1**(A10-3, A11-1)
- 整理効果1回につき **整理トークン+1**(A10-5, A11-1)
- 情熱トークンはゲーム開始時 **3** から開始する(A10-4)。
- 疲労効果は情熱トークンを1減らす。0未満にはならない(A10-4)。
- いずれのトークンも上限なし(A11-7)。

### 8.2 消費(講習期教室会議フェーズで実行、A11-3)

| 消費トークン | 操作 | 効果 | 消費量 |
|---|---|---|---|
| 情熱 | SR復活 | 任意の裏返しSR 1枚を表向きに戻す | 情熱-1 |
| 情熱 | SSR復活 | 任意の裏返しSSR 1枚を表向きに戻す | 情熱-3 |
| 発想 | カード追加獲得 | 任意の8山札から1山を選び、トップ3枚から1枚選んで任意デッキに追加 | 発想-1 |

## 9. ゲーム終了処理とスコア計算

1. 後期4日目(ターン13)の教室会議フェーズが終了した時点でゲーム終了。
2. 通常期+講習期で累積したステータスから、**動員 / 退塾 / 入退差 / 満足** の4観点で最終評価する。

## 15. カード移動規則

### 15.1 通常期 研修フェーズ

```
8山札(カテゴリ別×{地域/全校})
  ├ ターンのpool指定(local/zenkou)に従い4山を対象
  ├ プレイヤーが各回カテゴリを1つ選び、その山のトップから1枚ドロー
  ├ 同カテゴリ連続選択可
  └ 4回ドロー → 手札4枚 (秘匿)
```

### 15.2 通常期 教室行動フェーズ

```
手札(4枚)
  ├ 3枚 → 各スタッフ枠(leader/teacher/staff) ※並行なら追加配置可
  └ 余り → 次フェーズで処理

未割当スタッフ:
  ← Nプール(nPool) から1枚選択 → スタッフ枠に配置 → 効果適用
  ※ 効果適用後、選んだNカードはnPoolから抜けて placed[staff] に入る
  ※ Nカードに職種制限はない。未割当スタッフであれば任意のスタッフへ割当可
  ※ nPoolが空の場合、未割当スタッフには何も割り当てない
  ※ おすすめ行動ボーナス(動員→体験+2, 庶務→経理+2)を適用
```

### 15.3 通常期 教室会議フェーズ

```
スタッフ枠(leader/teacher/staff の placed配列)
  → 該当スタッフ別講習デッキ(staffDecks.{leader|teacher|staff})へ移動
  ※ Nカードも含めて全て移動 (Nプールに戻らない)

余りカード(手札に残った1枚以上):
  ├ R/SR かつ 講師可 のいずれか1枚 → アルバイト講師デッキ(任意)
  │     └ 既存があれば旧カードを除外
  └ 他の余り(または条件不一致) → ゲーム除外
```

### 15.4 講習期 準備フェーズ(前期1日目開始前 / 後期1日目開始前)

```
8山札(継続使用)
  → 1山選択 → トップ3枚を見る
  → 3枚から1枚選択 → 「新カード」
  → 残り2枚は当該山の捨て札へ

新カード:
  ├ プレイヤーが交換を実行:
  │   新カード → 4デッキ(staffDecks.leader/teacher/staff/alba)のいずれかへ追加
  │   交換相手(=同レアリティ以下の既存カード1枚) → ゲーム除外
  └ プレイヤーが交換を放棄:
      新カード → ゲーム除外

回数:
  前期1日目開始前: 3回連続
  後期1日目開始前: 2回連続
```

### 15.5 講習期 教室行動フェーズ

```
4デッキ(staffDecks.{leader|teacher|staff|alba})
  → 各デッキから表向きカード1枚を選択(計4枚/ターン)
  → 並行効果カードは追加配置可
  → placed.{staff} へ → 効果解決
  ※ おすすめ行動ボーナス(教務→入塾+2, 応対→満足+2)を適用
  ※ 使用後の SR/SSR → staffFlipped.{staff} に instanceId 登録(裏返し)
  ※ R/N → デッキ内残留(再使用可)
```

### 15.6 講習期 教室会議フェーズ

```
各スタッフ:
  ├ staffRestActivity[staff] === false (このターン1枚も使わなかった):
  │     └ SR復活: staffFlipped[staff] からSRをすべて削除(表向きへ)

  └ ターン=8 (中期2日目)で、staffMidRestRecord[staff].mid1 === true かつ .mid2 === true:

```

## 16. 講習期 SR/SSR 状態管理

### 16.1 裏返し管理

- 同名SR/SSRの個別管理のため、デッキ追加時に `instanceId`(uuid等)を付与する。
- `staffFlipped.{leader|teacher|staff|alba}` は `Set<instanceId>`。
- カード使用時:
  ```javascript
  if (card.rarity === 'SR' || card.rarity === 'SSR') {
    gameState.staffFlipped[staff].add(card.instanceId);
  }
  ```
- 配置選択時のフィルタ: `!staffFlipped[staff].has(card.instanceId)`

### 16.2 復活ロジック

```javascript
function checkRevival(turnIdx) {
  const config = TurnManager.TURN_CONFIG[turnIdx];
  if (config.period !== 'summer') return;

  for (const staff of ['leader', 'teacher', 'staff', 'alba']) {
    // SR復活: そのターン休みなら、デッキ内全SRを復活
    if (!gameState.staffRestActivity[staff]) {
      const deck = gameState.staffDecks[staff];
      for (const card of (deck || [])) {
        if (card.rarity === 'SR') {
          gameState.staffFlipped[staff].delete(card.instanceId);
        }
      }
    }

    // 中期record更新
    if (config.subPeriod === 'mid') {
      const slot = (turnIdx === 7) ? 'mid1' : 'mid2';
      gameState.staffMidRestRecord[staff][slot] = !gameState.staffRestActivity[staff];
    }

    // SSR復活: 中期2日目(turnIdx===8)で、mid1とmid2両方 true
    if (config.ssrRevival) {
      const rec = gameState.staffMidRestRecord[staff];
      if (rec.mid1 === true && rec.mid2 === true) {
        const deck = gameState.staffDecks[staff];
        for (const card of (deck || [])) {
          if (card.rarity === 'SSR') {
            gameState.staffFlipped[staff].delete(card.instanceId);
          }
        }
      }
    }
  }
}
```

## 18. バリデーション規則(計算機モード継承前提)

計算機モードを夏期でも実装する(A13-1)。以下のバリデーションが必要。

### 18.1 通常期 研修フェーズの入力検証

- 引くカテゴリはプレイヤーが選択。プールは仕様で確定(local/zenkou)。
- 入力するカードNo: そのカテゴリ・プールに存在する R/SR/SSR(レアリティはターンと月で確定)。
- 同じ山に存在しないカードNoはエラー。
- 同名カードの手札重複は無制限(A4-6 推奨どおり)。

### 18.2 通常期 教室行動フェーズの入力検証

- 配置先スタッフは職種制限(【】)を満たす。
- 手札に存在するカードのみ配置可。
- 同一スタッフへの複数枚配置は **並行効果カード** のみ可。
- 未割当スタッフへのNカード選択は `nPool` 内に存在するカードのみ可。
- Nカードは職種制限なしとして扱う。`nPool` が空の場合、未割当スタッフへの補完入力は不要。

### 18.3 通常期 教室会議フェーズの入力検証

- アルバイト講師デッキへの移動候補は R/SR かつ 講師可カードのみ。

### 18.4 講習期 準備フェーズの入力検証

- トップ3枚から1枚選ぶUI。
- 交換相手のカードは新カードと同一レアリティ以下のみ。
- 4デッキのいずれにも該当カードがない場合: 「交換せず除外」のみ選択可。

### 18.5 講習期 教室行動フェーズの入力検証

- 配置するカードは該当スタッフのデッキ内に存在し、表向き(裏返しでない)であること。
- 各デッキから1枚が基本、並行効果カードのみ追加配置可。
- 「休む」を明示選択した場合、そのスタッフのターン使用はゼロとして `staffRestActivity[staff] = false` を確定。

## 19. スコア計算アルゴリズム

### 19.1 春期スコアリングとの差分

| 項目 | 春期 | 夏期(案) |
|---|---|---|
| ターン数 | 8 | 13(通常期4 + 講習期9) |
| 評価指標 | 動員 / 退塾 / 入退差 / 満足 | 同じ(通常期+講習期の累積ステータスから算出) |
| ランクCSV | rankFresh / rankPro | rankSummerFresh / rankSummerPro(新規、A12-2) |
| 1ターン当たりカード使用枚数 | 3スタッフ × 1 = 3 | 通常期 3 + 講習期 4(並行カードで増加可) |
| おすすめ行動ボーナス | カテゴリ別+1/枚 | カテゴリ別+2/枚 |
| 講習期トークン | なし | PRO限定で情熱/発想/整理を講習期に消費 |

### 19.2 計算式(春期準拠の擬似コード)

```javascript
function calcSummerScore(player, difficulty) {
  const rankTable = loadRankCsv(difficulty === 'fresh' ? 'rankSummerFresh' : 'rankSummerPro');
  const taisyutsu = Math.max(0, 15 - player.satisfaction) + Math.max(0, 15 - player.accounting);
  const enrollmentDiff = player.enrollment - taisyutsu;
  const scores = {
    mobilization:   evalThreshold(rankTable, 'experience', player.experience),
    withdrawal:     evalThreshold(rankTable, 'withdrawal', taisyutsu),
    enrollmentDiff: evalThreshold(rankTable, 'enrollmentDiff', enrollmentDiff),
    satisfaction:   evalThreshold(rankTable, 'satisfaction', player.satisfaction),
  };
  const totalScore = sumScores(scores);
  const grade = lookupGrade(rankTable, totalScore);
  const title = difficulty === 'pro' ? lookupTitle(rankTable, grade) : null;
  return { scores, totalScore, grade, title };
}
```
