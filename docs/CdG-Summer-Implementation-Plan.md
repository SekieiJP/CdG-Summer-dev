# カードで学習塾 夏期講習編 実装計画

> 入力仕様: `docs/CdG-Summer-Spec.md` v0.3
> 関連方針: `docs/CdG-Summer-Repository-Policy.md`
> 目的: 春期講習編の資産を参照しつつ、夏期講習編として独立実装するための作業順序と検証項目を定義する。

---

## 1. 実装原則

- `SekieiJP/CdG-Summer-dev` は夏期講習編の成果物だけを管理する。
- 春期の `game/`, `solver/`, `gas/`, `tests/` は参照元とし、そのまま公開しない。
- ブラウザ版はソロ専用で実装する。ルールブック上の1〜6人プレイは紙面ルールとして扱う。
- まず全体フローを確認できるUIモックを固め、その後にゲームロジックを段階実装する。
- 実装は小さい単位で進め、各フェーズごとに保存/復元とE2E確認を行う。

## 3. モジュール計画

### 3.1 既存春期モジュールからコピーして改修

| 春期由来 | 夏期での扱い |
|---|---|
| `cardManager.js` | CSV読込、効果パーサーを流用。8山札分類と夏期特殊効果ハンドラを追加 |
| `gameState.js` | `trainingPools`, `nPool`, `staffDecks`, `staffFlipped`, `staffMidRestRecord`, `tokens` を追加 |

| `uiController.js` | 通常期/講習期の画面を分け、スタッフ別デッキ表示モードを追加 |
| `logger.js` | 講習期トークン獲得/消費、復活、交換ログを追加 |

### 3.2 夏期独自モジュール

| モジュール | 責務 |
|---|---|

| `summer/staffDeck.js` | スタッフ別講習デッキ、Nカード移動、アルバイト講師1枚スロット |
| `summer/summerPhaseManager.js` | 前期3回/後期2回の準備フェーズと13ターン遷移 |
| `summer/flipManager.js` | SR/SSR裏返し、毎ターンSR復活、中期2日休みSSR復活 |
| `summer/summerTokenManager.js` | 情熱/発想/整理トークンの獲得、消費、残量管理 |
| `summer/recommendBonus.js` | 通常期:動員/庶務、講習期:教務/応対の+2加算 |

## 5. UI実装順序

1. タイトル/難易度/計算機モード選択
2. 13ターンタイムライン
3. 通常期研修: 地域/全校、4カテゴリ、手札4枚
4. 通常期教室行動: 3スタッフ枠、Nプール、配置確定
5. 通常期教室会議: スタッフ別講習デッキ、アルバイト講師候補
6. 講習期準備: 8山札→トップ3枚→1枚選択→交換
7. 講習期教室行動: 4デッキ、休むボタン、裏返し表示
8. 講習期教室会議: SR/SSR復活、PROトークン消費
9. 結果画面: 4指標、退塾計算、ランク表示

スタッフ別デッキは、仕様どおりタブ表示モードと全体表示モードを切り替えられるようにする。

## 6. 検証計画

### 6.1 単体テスト

- 8山札の構成とSR半数分配
- 同カテゴリ連続選択を含む通常期4回ドロー
- Nプール消費とスタッフ別講習デッキへの移動
- アルバイト講師デッキのR/SRかつ講師可判定
- 講習期準備のトップ3枚選択と同一レアリティ以下交換
- SR/SSR裏返しと復活条件
- 講習期トークンの獲得/消費/上限なし
- おすすめ行動ボーナスの期別カテゴリ判定

### 6.2 E2Eテスト

- FRESHで通常期4ターンを完走
- FRESHで講習期9ターンを完走
- 中期1日目/2日目の両方休みでSSRが復活
- 片方だけ休みではSSRが復活しない
- PROで情熱/発想/整理トークンを獲得し、講習期会議で消費
- LocalStorage保存後にリロードして同じフェーズへ復帰
- 結果画面でランクCSVが読み込まれる

## 7. 実装前ゲート

| ID | 必要な判断 |
|---|---|
| Q12-3 | `rankSummerFresh.csv` / `rankSummerPro.csv` の具体閾値 |
| Q5-9 | Nプール5枚が枯渇した後の未割当スタッフ処理 |
| Q5-10 | Nカードを任意スタッフへ制限なく割り当ててよいか |

この3点は、M6までに確定が必要。M3までは推奨案を仮置きしてUI/ロジックの分岐を残す。

## 8. 初回実装タスク

1. `public/index.html` のフローモックで全13ターンと各フェーズの画面構成を確認する。
2. 夏期用 `game/` を新規作成し、春期ファイルを直接公開しない形で最小HTML/JS/CSSを置く。
3. `trainingPool.js` と `TURN_CONFIG` を先に実装し、通常期4ターンの状態遷移を作る。
4. スタッフ別講習デッキとNプールを実装する。
5. 講習期準備とSR/SSR裏返しを実装する。
6. PROトークン、保存、スコア、E2Eを追加する。
