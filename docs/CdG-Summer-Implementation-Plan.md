# カードで学習塾 夏期講習編 実装計画

> 入力仕様: `docs/CdG-Summer-Spec.md` v0.4
> 関連方針: `docs/CdG-Summer-Repository-Policy.md`
> 目的: 春期講習編の資産を参照しつつ、夏期講習編として独立実装するための作業順序と検証項目を定義する。

---

## 1. 実装原則

- `SekieiJP/CdG-Summer-dev` は夏期講習編の成果物だけを管理する。
- 春期の `game/`, `solver/`, `gas/`, `tests/` は参照元とし、そのまま公開しない。
- ブラウザ版はソロ専用で実装する。ルールブック上の1〜6人プレイは紙面ルールとして扱う。
- まず全体フローを確認できるUIモックを固め、その後にゲームロジックを段階実装する。
- 実装は小さい単位で進め、各フェーズごとに保存/復元とE2E確認を行う。
- UIは春期講習編の見た目と情報密度を基準にし、常時表示ではなく「必要な時だけ開く」導線を優先する。
- 春期CSS、カテゴリ/レアリティのイメージカラー、カード効果発動演出、ステータスのプログレスゲージは継承して利用する。

## 3. モジュール計画

### 3.1 既存春期モジュールからコピーして改修

| 春期由来 | 夏期での扱い |
|---|---|
| `cardManager.js` | CSV読込、効果パーサーを流用。8山札分類と夏期特殊効果ハンドラを追加 |
| `gameState.js` | `trainingPools`, `nPool`, `staffDecks`, `staffFlipped`, `staffMidRestRecord`, `tokens` を追加 |
| `turnManager.js` | 13ターン構成と `summerPrep` / `summerAction` / `summerMeeting` を追加 |
| `scoreManager.js` | `rankSummerFresh.csv` / `rankSummerPro.csv` を参照 |
| `saveManager.js` | 夏期追加状態、Setの配列化、カード順の保存を追加 |
| `uiController.js` | 通常期/講習期の画面を分け、春期の効果発動演出・プログレスゲージ・スタッフ別デッキ表示モードを継承 |
| `logger.js` | 講習期トークン獲得/消費、復活、交換ログを追加 |

### 3.2 夏期独自モジュール

| モジュール | 責務 |
|---|---|
| `summer/trainingPool.js` | 4カテゴリ×地域/全校の8山札構成、SR半数分配、捨て札再構成 |
| `summer/staffDeck.js` | スタッフ別講習デッキ、Nカード移動、アルバイト講師1枚スロット |
| `summer/summerPhaseManager.js` | 前期3回/後期2回の準備フェーズと13ターン遷移 |
| `summer/flipManager.js` | SR/SSR裏返し、毎ターンSR復活、中期2日休みSSR復活 |
| `summer/summerTokenManager.js` | 情熱/発想/整理トークンの獲得、消費、残量管理 |
| `summer/recommendBonus.js` | 通常期:動員/庶務、講習期:教務/応対の+2加算 |

## 5. UI実装順序

1. 春期講習編のCSSとUIアセットを取り込み、カテゴリ/レアリティ色、カード効果発動演出、ステータスプログレスゲージを継承する。
2. 春期講習編準拠のヘッダー、ステータス、手札4枠の骨格を固定する。
3. ヘッダーのターン情報タップで現在フェーズ詳細を開く導線を実装する。
4. ヘッダーの絵文字メニューから、現在のデッキと今後のスケジュールをオーバーレイ表示する。
5. 13ターンタイムラインと開発用フェーズ切替を実装する。
6. 通常期研修: 地域/全校、4カテゴリ、手札4枚、四択ボタン4個を1画面に収める。
7. 通常期研修: 獲得カードを手札へスライドイン表示し、そのまま次の四択へ進める。
8. 通常期教室行動: 3スタッフ枠、Nプール、配置確定。縦画面では4枠を2列×2行で表示する。
9. 通常期教室会議: スタッフ別講習デッキ、アルバイト講師候補を整理する。
10. 講習期準備: 8山札→トップ3枚→1枚選択→交換。
11. 講習期教室行動: 4デッキ、休むボタン、裏返し表示。
12. 講習期教室会議: SR/SSR復活、PROトークン消費。
13. 結果画面: 4指標、退塾計算、ランク表示。

## 6. 検証計画

### 6.1 単体テスト

- 8山札の構成とSR半数分配
- 同カテゴリ連続選択を含む通常期4回ドロー
- Nプール消費とスタッフ別講習デッキへの移動
- Nプール枯渇後は未割当スタッフへ何も割り当てない処理
- Nカードに職種制限がなく、任意の未割当スタッフへ割り当てられること
- アルバイト講師デッキのR/SRかつ講師可判定
- 講習期準備のトップ3枚選択と同一レアリティ以下交換
- SR/SSR裏返しと復活条件
- 講習期トークンの獲得/消費/上限なし
- おすすめ行動ボーナスの期別カテゴリ判定

### 6.2 E2Eテスト

- 初期表示ではフェーズ詳細オーバーレイが閉じている
- ヘッダーのターン情報タップで現在フェーズ詳細が開閉する
- ヘッダーメニューで現在のデッキと今後のスケジュールが開閉する
- 通常期研修の四択4ボタンが1画面内に収まり、獲得カードが手札へ表示されたまま次の選択を継続できる
- 縦画面で4スタッフ枠が2列×2行になる
- FRESHで通常期4ターンを完走
- FRESHで講習期9ターンを完走
- 中期1日目/2日目の両方休みでSSRが復活
- 片方だけ休みではSSRが復活しない
- PROで情熱/発想/整理トークンを獲得し、講習期会議で消費
- 春期CSS由来のカテゴリ/レアリティ色、カード効果発動演出、ステータスプログレスゲージが表示される
- LocalStorage保存後にリロードして同じフェーズへ復帰
- 結果画面でランクCSVが読み込まれる

## 8. 初回実装タスク

1. `public/index.html` のフローモックで、常時表示パネルを減らした導線と全13ターンの画面構成を確認する。
2. 夏期用 `game/` を新規作成し、春期ファイルを直接公開しない形で最小HTML/JS/CSSを置く。
3. 春期CSSとUIアセットをファイル単位でコピーし、カテゴリ/レアリティ色、カード効果発動演出、ステータスプログレスゲージを夏期画面で再利用する。
4. `trainingPool.js` と `TURN_CONFIG` を先に実装し、通常期4ターンの状態遷移を作る。
5. 通常期四択と獲得カードスライドインを先に実装し、1画面完結の操作感を固定する。
6. スタッフ別講習デッキとNプールを実装する。N枯渇後は未割当スタッフへ何も割り当てず、Nカードの職種制限は設けない。
7. 講習期準備とSR/SSR裏返しを実装する。
8. PROトークン、保存、スコア、E2Eを追加する。
